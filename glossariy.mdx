Объяснение всех ключевых терминов и аббревиатур, используемых в Chat Bridge.

## А

**API (Application Programming Interface)**
Набор команд, которые может выполнять система. Например, "получить список чатов" — это одна из команд API.

**API Token** (также: Access Token)
Уникальный пропуск для доступа к системе. Выглядит как длинная строка символов (вроде: `abcd1234efgh5678`). Нужен для всех операций в системе.

**Avito**
Крупнейший портал недвижимости в России. На нём размещают объявления риэлторы.

## В

**Верификация**
Процесс подтверждения личности через SMS код. SMS код подтверждает, что это действительно ваш номер телефона.

## Г

**Геолокация**
GPS координаты объекта (широта и долгота). Используются для отображения объекта на карте.

## Д

**Domclick**
Платформа недвижимости от Яндекса. На ней можно размещать объявления и искать недвижимость.

## З

**Зашифрованный**
Способ сохранения информации, при котором она неразборчива для человека, но доступна системе. Например, все токены платформ зашифрованы в базе данных.

## И

**Интеграция**
Способ соединения двух систем так, чтобы они обменивались данными. Например, интеграция с Avito означает, что Chat Bridge может получать данные из Avito.

**ИНН (Идентификационный номер налогоплательщика)**
Уникальный номер организации или ИП. В России состоит из 10 (для ИП) или 12 (для организаций) цифр.

## К

**Cian**
Популярный портал объявлений о продаже и сдаче недвижимости в России. Конкурент Avito.

**client_secret**
API токен для доступа к Cian API. Это как пароль для системы.

**Chat (Чат)**
Переписка между менеджером и клиентом по одному объявлению.

## М

**Magic Link** (Пригласительная ссылка)
Временная ссылка для присоединения к компании. Когда человек нажимает на ссылку, он автоматически добавляется в компанию.

**MAINTAINER**
Роль владельца/администратора компании. Имеет полный доступ к системе.

**MANAGER**
Роль рядового менеджера. Видит только свои объявления и чаты.

## О

**OAuth**
Способ авторизации (вход в систему) через другой сервис. Например, "Войти через Avito" — это OAuth авторизация.

**ОГРН (Основной государственный реестровый номер)**
Уникальный номер организации. В России состоит из 15 цифр.

## П

**Платформа**
Веб-сайт или приложение, на котором размещаются объявления. Например, Avito, Cian, Domclick.

**Proxy (Прокси)**
Промежуточный сервер, который передаёт запросы. Используется для безопасности.

## Р

**RESPONSIBLE**
Роль ответственного/супервайзера. Видит все объявления и чаты, но не может управлять интеграциями.

**Refresh Token**
Специальный токен для обновления access_token'а. Когда access_token истекает, используется refresh_token для получения нового.

## С

**SMS**
Текстовое сообщение на телефон. Используется для отправки кода верификации.

**Синхронизация**
Процесс загрузки и обновления данных из внешних систем (Avito, Cian, Domclick) в Chat Bridge.

**stats_secret**
API токен для доступа к статистике Domclick API.

**chats_secret**
API токен для доступа к чатам Domclick API.

## Т

**Токен** (также: API Token, API Key)
Уникальный ключ для доступа к системе или API. Похож на пароль, но используется для программ, а не для людей.

## Ф

**Фид**
XML файл со списком объявлений. Риэлтор создаёт фид на своём сайте, и система Chat Bridge загружает из него объявления.

## Х

**Hash (Хеш)**
Результат криптографической функции. Используется для безопасности данных.

## Ц

**Целевая аудитория**
Люди, для которых предназначена система. Для Chat Bridge — агентства недвижимости и риэлторы.

## Ч

**Чат**
Переписка между двумя людьми. В контексте Chat Bridge — переписка между менеджером и клиентом.

## Э

**Endpoint** (Эндпоинт)
Одна конкретная команда API. Например, "получить список чатов" — это один endpoint.

**External ID**
ID сущности в другой системе (например, ID чата на Avito).

## Ю

**ULID**
Уникальный идентификатор, используемый в системе вместо стандартного числового ID. Выглядит как: `01ARZ3NDEKTSV4RRFFQ69G5FAV`.

## Роли и статусы

### Роли в компании
- **MAINTAINER** — владелец (полный доступ)
- **RESPONSIBLE** — ответственный (просмотр, управление ссылками)
- **MANAGER** — менеджер (только свои данные)

### Статусы компании
- **NEW** — новая компания
- **WAITING_FOR_PROVIDER_SELECTION** — выбор платформ
- **WAITING_FOR_AVITO_FEED** — ввод фида Avito
- **WAITING_FOR_AVITO_ACCESS** — OAuth авторизация в Avito
- **WAITING_FOR_CIAN_ACCESS** — ввод токена Cian
- **WAITING_FOR_DOMCLICK_ACCESS** — ввод токенов Domclick
- **WAITING_FOR_FULL_SYNCHRONIZATION** — синхронизация
- **COMPLETED** — готова к работе

### Статусы сообщений
- **PENDING** — в процессе отправки
- **SENT** — отправлено
- **DELIVERED** — доставлено
- **READ** — прочитано

## Связанные термины

**Объявление vs Чат vs Сообщение**
- **Объявление** = само предложение (квартира за 50 000 рублей)
- **Чат** = переписка с одним клиентом по одному объявлению
- **Сообщение** = одно отдельное сообщение в чате

**Авторизация vs Аутентификация**
- **Аутентификация** = подтверждение: "ты точно тот, кем заявляешь?" (вход по SMS коду)
- **Авторизация** = разрешение: "ты имеешь право делать это?" (роль MANAGER может видеть только свои чаты)

**Access Token vs Refresh Token**
- **Access Token** = основной ключ для работы (действует ~1 год для Avito)
- **Refresh Token** = ключ для обновления access_token'а, когда тот истекает

**Token vs Password**
- **Token** (API Token) = автоматический пропуск для систем
- **Password** (Пароль) = ручной пропуск для людей
